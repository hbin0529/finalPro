<b>59'HOUSE(오구싶은집)</b>은 가정에서 사용할 가구 및 인테리어 소품 판매를 주 목적으로 하는 E-Commerce 쇼핑몰 사이트입니다.<br>
또한 자신의 집 인테리어를 소개하고 정보를 공유하며 소비자의 구매욕구를 불러일으키는 커뮤니티를 겸한 통합 플랫폼입니다.

## :ticket: 주요기능 
### :one: 메인 페이지
* Ajax를 통한 카테고리별 높은 조회수의 상품 나타내기
### :two: 로그인 및 회원가입 
* 카카오 소셜 로그인 API, 다음 우편 주소 API 이용하여 간편한 가입 지원 / bcrypt를 통한 패스워드 암호화
### :three: 집들이
* Summernote 라이브러리 활용한 게시글 작성 / 구매한 물품을 이용하여 인테리어 경험 공유 / 인테리어 업체 별 인테리어 후기, 예시 확인
### :four: 구매 및 판매 
* 카테고리와 상품 옵션으로 구분하여 상품 등록 / 상품 별 구매자 문의, 판매자 답변 기능 / 구매자 리뷰 및 별점  기능  / 재고  관리 / 장바구니 담기 기능 / 판매자 물품 판매 통계 기록 확인 가능
### :five: 충전
* Ajax와 MyBatis를 통한 포인트 충전하기
### :six: 마이페이지
* 구매자 마이페이지에서 자신이 구매한 물품 조회 / 확정된 주문에 리뷰 작성 가능 / 판매자 마이페이지에서 문의에 대한 답변 및 주문에 대한 정보 확인 후 취소, 판매 확정 신청

## :pushpin: 마치며
 <b>프로젝트는 하나의 퍼즐놀이 같았습니다.</b> 교육기관에서 배웠던 것들과, 궁금하여 더 찾아보았던 기술들을 하나의 퍼즐 조각처럼 응용하여 내가 만들고자 하였던 웹 사이트를 팀원들과 함께 구현해 낸다는 것이 가장 흥미로웠습니다.

<b>[프로젝트 진행하면서 어려웠던 점]</b><br>
 프로젝트를 진행하면서 가장 어려웠던 점은 메인 페이지에 불러올 SQL 쿼리문을 작성하는 것이었습니다. 정보가 카테고리별로 보이도록, 높은 조회수 순으로 정렬되도록 여러가지 방법으로 배운 것들을 응용해 보았지만 지속적인 오류가 났습니다. 그래서 저는 벤치마킹 할 만한 다른 예시는 없는지, 이런 기능을 구현할 땐 어떤 함수가 적절한지 다방면으로 알아보았고 해당 정보를 바탕으로 여러가지 시도를 해보았습니다. 그 결과, 오류가 수정되어 기능이 정상적으로 작동되었고, 동시에 작은 희열감도 느낄 수 있었습니다. 이러한 경험을 바탕으로, 저는 해결하기 어려운 문제라도 끈질기게 노력하다 보면 길이 보인다는 깨달음을 얻을 수 있었습니다.

<b>[협업 시 소통의 중요성을 배우다]</b><br>
 혼자 진행하는 세미 프로젝트와는 달리, 여러 명이 협력하여 진행하는 파이널 프로젝트는 소통이 중요했습니다. 저의 아이디어와 생각을 조원들과 공유해야 했고, 기능 구현 부분에 대해 세세한 논의가 필요했습니다. 저를 비롯한 팀원 모두가 긍정적이고 적극적으로 커뮤니케이션에 임했고, 자신이 맡은 부분에 대해서 책임감 있게 프로젝트를 수행하였습니다. 서로의 코드를 공유해서 조언도 하고 도움도 받으며 주어진 일정과 순서에 따라 성실히 기능을 구현하였습니다. 그 결과, 내심 걱정했던 처음과 달리 기대보다 더 완성도 높은 결과물을 도출해 낼 수 있었습니다. 프로젝트를 진행할 때 협업자 간의 의사소통이 얼마나 중요한지 깨달은 좋은 경험이었습니다.

<b>[프로젝트 경험을 통해 세운 앞으로의 목표]</b><br>
 처음 구상했던 기능들 외에 조금 더 욕심을 내서 아임포트 API를 이용한 상품 결제, 환불 부분을 구현해 보고자 하였으나, 진행하지 못한 아쉬움이 남습니다. 진행에 필요한 정보는 모두 작업해 놓았으나 기능까지 구현하기에는 아직 저의 개발 지식이 부족했습니다. 그래서 저는 부족한 CS 이론을 보완하기 위해 관련 서적들을 꾸준히 읽고자 하며, 웹 개발의 ‘봄’ 스프링에 대해 인프런 강의를 듣으며 실력을 증진하고자 합니다.
